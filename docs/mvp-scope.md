# MVP Scope

## MVP Goal

Validate whether users understand and value a nightlife recommendation product that suggests places based on personal preferences, location, music taste, and vibe.

## Core MVP Features

### 1. Quick Preference Profile

The user answers a short set of questions to define what kind of night they are looking for.

Possible inputs:

- Music preference
- Preferred atmosphere
- Budget range
- Distance preference
- Dress code comfort
- Type of venue
- Group or solo context

### 2. Location-based Recommendations

The product recommends nightlife spots within a practical radius, initially around 15 km.

### 3. Venue Cards

Each recommendation should show the most important information quickly.

Recommended information:

- Venue name
- Type of place
- Music style
- Location
- Distance
- Price level
- Dress code
- Opening hours
- Rating
- Visual preview
- Short vibe description

### 4. Strong Visual Preview

Users should understand what to expect before choosing a place. Visuals should communicate atmosphere, crowd, dress code, and energy.

### 5. Recommendation and Rating System

Ratings and recommendations are important for trust. The first version can start simple, but the product should clearly show why a place is recommended.

## Out of Scope for First MVP

These features are not priority for the first version:

- Direct ticket sales
- Complex group matching
- Full social network features
- Chat between users
- Advanced event organizer tools
- International expansion
- Complex AI automation

## Success Criteria

The MVP is successful if test users can quickly understand:

1. What the product does
2. Why a venue is recommended
3. Whether they would use it before going out
4. What information is missing before making a decision
